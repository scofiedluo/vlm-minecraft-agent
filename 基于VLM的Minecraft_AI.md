# 代码测试：基于 VLM 的 Minecraft AI

## 一、测试背景

本测试旨在使用 **视觉语言模型（VLM）** 构建 Minecraft 游戏 AI，主要考察工程实践能力和对 VLM 的理解和使用。Minecraft 作为一款开放世界沙盒游戏，具备丰富的视觉场景、复杂的生存逻辑和高度自由的操作空间，是检验多模态模型理解与决策能力的理想环境。

---

## 二、测试任务

### 核心目标

使用 **任意 VLM** 作为决策核心，构建一个能够在 Minecraft 中自主运行的 AI Agent。该 Agent 需要：

1. **视觉理解**：通过屏幕截图/视频流理解游戏画面（地形、方块类型、生物、物品栏、生命值等）
2. **决策推理**：基于视觉信息和游戏状态做出生存决策（采集资源、合成物品、建造、战斗、探索等）
3. **动作执行**：将决策转化为游戏操作指令（键鼠模拟或 Mod API 调用）

### 可使用的辅助工具

- **任意 Screen Capture 工具**
- **任意 VLM API 或开源模型**
- **任意 Vibe Coding 手段**
- **任意辅助库**（如 MineRL、Malmo、mineflayer 等 Minecraft 交互框架）

---

## 三、技术要求

| 编号 | 要求 | 说明 |
|------|------|------|
| B1 | VLM 接入 | 成功接入至少一个 VLM 模型，实现屏幕图像输入 → 动作输出 的完整链路 |
| B2 | 视觉理解 | AI 能够识别游戏中的关键元素：地形类型（草地、水、岩石）、方块种类、生物（怪物/动物）、物品栏内容、生命值/饥饿值、昼夜状态等 |
| B3 | 决策输出 | AI 能够根据视觉信息输出明确的动作指令（移动方向、挖掘方块、放置方块、攻击、合成、使用物品等） |
| B4 | 动作执行 | 决策指令能够实际作用于游戏（通过键鼠模拟、Mod API 或 mineflayer 等工具） |
| B5 | Demo | 提交一个游戏 Demo 录屏（至少 3 分钟），演示 AI 可在其中自主运行并完成基本生存任务 |
